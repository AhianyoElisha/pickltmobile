# Screen Spec: Authentication (Login, Signup, OTP)

## What is this page about?
[cite_start]The screens where users create a new account or log into an existing one, including the mandatory phone verification step[cite: 81].

## User Flows
* [cite_start]User chooses to sign up via custom email or Google[cite: 82].
* [cite_start]User enters credentials and submits[cite: 82].
* [cite_start]User is redirected to the OTP verification screen[cite: 82].
* [cite_start]User receives an SMS, enters the OTP, and confirms[cite: 82].
* [cite_start]On success, new users route to the Profile Confirmation screen; returning users route to Home[cite: 82].

## Data Displayed
* [cite_start]Input fields for Email, Password[cite: 83].
* [cite_start]Social Auth buttons (Google)[cite: 83].
* [cite_start]OTP input fields (typically 4-6 digits)[cite: 83].

## Components Needed
* [cite_start]Text Input fields (Standard and Password with visibility toggle)[cite: 84].
* [cite_start]Social Login Button (with Google icon)[cite: 84].
* [cite_start]Primary Action Buttons[cite: 84].
* [cite_start]OTP Input Component[cite: 84].

## States and Edge Cases
* [cite_start]**States:** Invalid email format, incorrect password, incorrect OTP, expired OTP[cite: 85].
* [cite_start]**Edge Case:** Resend OTP countdown timer active/inactive[cite: 85].
