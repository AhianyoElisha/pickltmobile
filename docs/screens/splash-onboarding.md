# Screen Spec: Splash & Onboarding

## What is this page about?
[cite_start]The initial launch experience that introduces the user to the PickLT app's value proposition before they log in or sign up[cite: 81].

## User Flows
* [cite_start]User opens the app and sees the Splash Screen[cite: 82].
* [cite_start]User swipes through a carousel of Onboarding Screens[cite: 82].
* [cite_start]User taps "Sign up with Email" or "Sign up with Google" or "Sign In" to navigate to the Authentication flow[cite: 82].

## Data Displayed
* [cite_start]App logo and branding[cite: 83].
* [cite_start]Illustrations/Images and brief marketing copy detailing instant/scheduled moves[cite: 83].

## Components Needed
* [cite_start]Carousel/Slider component with Skip and Next buttons[cite: 84].
* [cite_start]Primary Button ("Sign Up with Email")[cite: 84].
* [cite_start]Secondary Button ("Sign up with google")[cite: 84].
* [cite_start]Text link ("Sign In")[cite: 84].

## States and Edge Cases
* [cite_start]**Edge Case:** User has already completed onboarding previously (should bypass directly to Auth or Home if token exists)[cite: 85].
