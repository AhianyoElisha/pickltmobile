# Screen Spec: Password Change Flow

## What is this page about?
[cite_start]The recovery flow for users who need to reset or change their password, utilizing OTP via email or phone[cite: 81].

## User Flows
* [cite_start]User taps "Forgot Password" from Login[cite: 82].
* [cite_start]User selects delivery method: Email or Phone[cite: 82].
* [cite_start]User receives and enters OTP[cite: 82].
* [cite_start]User enters and confirms new password[cite: 82].
* [cite_start]User is redirected back to Login[cite: 82].

## Data Displayed
* [cite_start]Selection toggles/cards for Email vs Phone[cite: 83].
* [cite_start]OTP input[cite: 83].
* [cite_start]New Password and Confirm Password fields[cite: 83].

## Components Needed
* [cite_start]Radio buttons or selectable cards[cite: 84].
* [cite_start]OTP Input Component[cite: 84].
* [cite_start]Password Input Component[cite: 84].

## States and Edge Cases
* [cite_start]**States:** Passwords do not match, password doesn't meet security requirements[cite: 85].
